Custom Memory Allocator

Efficient memory management using mmap and custom functions in C

Project Overview

This project demonstrates a custom memory allocator implemented in C using the mmap system call. It provides custom memory management functions including myalloc(), myfree(), and myrealloc(). Instead of using standard C memory allocation functions like malloc() or free(), this project manages memory by reserving a fixed-size pool and handling memory blocks manually through a linked list of headers.

How It Works

  • Memory Pool: A block of memory is obtained using the mmap system call (default size: 1MB).
  • Custom Allocation: The memory is divided into chunks that are tracked using a linked list with headers.
  • Memory Management: Functions like myalloc() allocate memory, myfree() deallocates memory, and myrealloc() resizes blocks.

Usage Instructions

  1. Compile the project using the following commands:
    2. gcc -o allocator src/allocator.c src/memory_utils.c main.c -I include
  2. Run the program:
    3. ./allocator
  3. Follow the interactive prompts to allocate, free, and reallocate memory dynamically.

File Structure

The project consists of the following files:

  • allocator.h: Contains the function declarations for myalloc(), myfree(), and myrealloc().
  • memory_utils.h: Declares utility functions for memory block management.
  • allocator.c: Implements the memory allocation, free, and reallocation logic.
  • memory_utils.c: Contains utility functions like aligning memory size, coalescing blocks, and managing the free list.
  • main.c: A sample interactive program that demonstrates how to use the custom allocator.

Features

  • Efficient memory management using mmap for obtaining memory directly from the OS.
  • Custom memory allocation, freeing, and reallocation using a linked list of memory blocks.
  • Interactive menu-driven program that demonstrates memory allocation and deallocation in real time.

Future Enhancements

  • Adding support for multi-threading with thread-safe memory allocation.
  • Implementing memory defragmentation techniques.
  • Customizable memory pool size at runtime.
